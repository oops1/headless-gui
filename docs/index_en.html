About

headless-gui — an off-screen GUI engine written in Go. Renders widgets to an RGBA buffer and returns only changed tiles (64×64 px). Window system-independent — output can be connected to anything: native windows Win32/Cocoa/X11, WebSocket, RDP session.

Module: github.com/oops1/headless-gui/v3

Dependencies: ebitengine/purego, golang.org/x/image, golang.org/x/sys. No CGO.

Package Structure

PackageDescription
engine/Render cycle, canvas, event dispatch, fonts
widget/All widgets, themes, XAML loader, Grid layout
widget/datagrid/DataGrid core: ObservableCollection, bindings, columns
widget/treeview/TreeView core: model, templates, render, input
output/Frame / DirtyTile types
window/Native window Win32/Cocoa/X11 (no CGO)

Quick Start

import (
    "image"
    "image/color"
    "github.com/oops1/headless-gui/v3/engine"
    "github.com/oops1/headless-gui/v3/widget"
)

eng := engine.New(1920, 1080, 30) // width, height, FPS

root := widget.NewPanel(color.RGBA{R: 30, G: 30, B: 46, A: 255})
root.SetBounds(image.Rect(0, 0, 1920, 1080))

btn := widget.NewWin10AccentButton("Войти")
btn.SetBounds(image.Rect(860, 500, 1060, 540))
btn.OnClick = func() { fmt.Println("Click!") }
root.AddChild(btn)

eng.SetRoot(root)
eng.Start()
defer eng.Stop()

for frame := range eng.Frames() {
    for _, tile := range frame.Tiles {
        sendToClient(tile) // tile.X, tile.Y, tile.W, tile.H, tile.Data
    }
}

import "github.com/oops1/headless-gui/v3/window"

win := window.New(eng, "Моё приложение")

    log.Fatal(err)
}

    btn.OnClick = func() { fmt.Println("OK!") }
}

eng.SetRoot(root)

Architecture

The engine runs in a loop at the specified FPS. On each frame:

  1. Input events (mouse, keyboard) are processed
  2. The widget tree is drawn to an internal RGBA buffer via DrawContext
  3. The buffer is split into 64×64 tiles. Changed tiles are sent to the Frames() channel

All coordinates are absolute pixels. Widgets are positioned via SetBounds(image.Rectangle) manually or through XAML containers (Grid, StackPanel, DockPanel, Canvas).

XAML Loader

Supports a subset of WPF XAML. Loading:

// From file
root, reg, err := widget.LoadUIFromXAMLFile("layout.xaml")

// From bytes
root, reg, err := widget.LoadUIFromXAML(data)

// With baseDir for relative paths (Image.Source)
root, reg, err := widget.LoadUIFromXAMLWithBase(data, "/assets")

Fonts (FontCache)

Fonts are registered through the engine. Default is built-in font (DejaVu-like). Default DPI is 96.

eng.RegisterFontFile("Roboto", "./fonts/Roboto-Regular.ttf")
eng.SetDPI(120)

In widgets, size is specified in FontSize (points). Default is DefaultFontSizePt = 10.0.

output.Frame

type Frame struct {
    Seq       uint64
    Timestamp time.Time
    Tiles     []DirtyTile
}

type DirtyTile struct {
    X, Y, W, H int
    Data        []byte // RGBA-пиксели
}

XAML-реестр расширений

Пользовательские XAML-теги extension

Позволяет регистрировать собственные виджеты для использования в XAML.

// Интерфейс для чтения атрибутов XAML-элемента
type XAMLAttrs interface {
    Attr(names ...string) string   // значение атрибута
    Tag() string                    // имя тега
    Text() string                   // внутренний текст
    ChildCount() int                // число дочерних тегов
    ChildAttrs(index int) XAMLAttrs // атрибуты дочернего тега
}

// Тип функции-строителя
type XAMLWidgetBuilder func(attrs XAMLAttrs) (Widget, error)

// Регистрация
widget.RegisterXAMLWidget("MyWidget", func(attrs XAMLAttrs) (Widget, error) {
    w := NewMyWidget()
    if v := attrs.Attr("Color"); v != "" {
        w.Color = parseColor(v)
    }
    return w, nil
})

// Unregister
widget.UnregisterXAMLWidget("MyWidget")

After registration, the tag <MyWidget Color="#FF0000"/> will be processed by your builder.
